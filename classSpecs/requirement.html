The Requirement Class Specification.

This class models all requirements in a major.

Xandra Best. Wednesday, May 15, 2013




CLASS Requirement

No superclass.

State variables:

protected string name
Name of requirement.

protected int needed
Number of courses needed of requirement.

protected int has
Number of courses had of requirement.

protected bool satisfied
Whether requirement is satisfied.

Constructors:

Requirement
Default constructor.

No arguments.

State change:
Requirement object is initialized using init with arguments n="", need =0.

Requirement
Partial constructor.

Arguments:

string str
Name of requirement.

State change:
Requirement object is initialized using init with arguments n=str, need =0.

Requirement
Regular constructor.

Arguments:

string str
Name of requirement.

int n
Number of valid courses needed.

State change:
Requirement object is initialized using init with arguments n=str, need =n.

Requirement
Copy constructor.

Arguments:

const Requirement c
Reference to a requirement.

State change:
Requirement object is initialized by copying values from c.

Methods:

init
Initializes instance of Requirement.

Arguments:

string n
Name of requirement.

int needed
Number of courses needed for requirement.

State change:
State variables are set as follows: name = n, needed = need, has =0, satisfied = false.

Return value:
No return value.

copy
Copies state variable values from one requirement to another.

Arguments:

const Requirement c
Reference to a Requirement.

State change:
State variables are changed to match those of c.

Return value:
No return value.

Requirement =
Assignment operator.

Arguments:

const Requirement c
Reference to a Requirement.

State change:
State variables are changed using copy.

Return value:
A requirement that is a copy of c.

incrementHas
Deal with a course that satisfies a requirement.

No arguments.

State change:
Has is incremented by one and satisfied is updated.

Return value:
No return value.

decrementHas
Opposite of incrementHas

No arguments.

State change:
Has is decremented by one and satisfied is updated.

Return value:
No return value.

string getName
Get name of requirement.

No arguments.

State change:
No state change.

Return value:
Return name.

int getNeeded
Get number of requirements needed.

No arguments.

State change:
No state change.

Return value:
Return needed.

int getHas
Get number of requirements had.

No arguments.

State change:
No state change.

Return value:
Return has.

bool getSatisfied
Check whether requirement is satisfied.

No arguments.

State change:
No state change.

Return value:
Return satisfied.

setName
Set name.

Arguments:

string str
New name.

State change:
Name is set to str.

Return value:
No return value.

setNeeded
Set number of requirements needed.

Arguments:

int n
New number needed.

State change:
Needed is set to n.

Return value:
No return value.

friend bool ==
Equal to operator.

Arguments:

const Requirement l
Reference to Requirement.

const Requirement r
Reference to Requirement.

State change:
No state change.

Return value:
Return true if requirements are identical.

friend bool !=
Not equal to operator.

Arguments:

Requirement l
Reference to Requirement.

Requirement r
Reference to Requirement.

State change:
No state change.

Return value:
Return true if requirements are not identical.

virtual void getData
Writes out data about requirements satisfied.

Arguments:

ostream os
Reference to ostream.

State change:
name, need, has, satisfied written out to os.

Return value:
os.

virtual void display
Couts data about requirements satisfied.

No arguments.

State change:
No state change.

Return value:
Couts given data.
